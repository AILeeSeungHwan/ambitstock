## 1. TeamCreate 선언문

포스팅을 새로 만들거나 기존 포스팅을 대량 수정할 때, Claude Code에서 아래 선언을 먼저 입력한다:

```text
포스팅 만들땐 TeamCreate로 에이전트 팀생성해줘.
에이전트 팀으로 프로젝트 코드를 병렬 리뷰하고, 문제를 수정하며,
에이전트 이용해서 계획승인모드로 각 팀원이 변경하기 전에 반드시 plan approval 받도록 해줘.
우리는 포스팅을 만드는 것이므로 각 팀원은 할당된 포스팅을 만들고 승인받은 뒤 생성하도록 해줘.
모든 팀원 모델은 Sonnet으로 설정해줘.
```

## 3. 작업 흐름

### 실행 모드

```bash
# 포스팅 생성/대량 수정 시 — plan approval 모드
claude

# 일반 유지보수 작업 시 — skip-permissions 모드
claude --dangerously-skip-permissions
```

### 절차

```
1. 주제 선정 & 검색의도/추천의도 정의
   └─ Editor Lead가 주제를 배분하고 검색의도/추천의도를 명시
2. plan approval 요청
   └─ Content Builder가 아래 내용을 포함한 계획서 제출:
      - 제목 (title)
      - slug
      - 포스팅 유형 (추천형/리뷰형/결말해석형/OTT정리형/비교형/시리즈가이드형/트렌드형)
      - 검색의도 또는 추천의도 (1문장)
      - 대상 독자 & 취향 축 (1~2문장)
      - H2 목록 (5~8개)
      - 이미지 계획 (장수, 각 이미지 설명, 저작권 안전 여부)
      - 광고 배치 계획 (위치, 개수)
      - 관련 포스팅 내부링크 계획
3. Editor Lead 승인
4. 초안 작성
   └─ 포스트 구조 표준 (POST_TEMPLATE.md) 준수
5. 검수 (병렬 가능)
   └─ SEO Reviewer: 검색의도/slug/제목/내부링크
   └─ Fact Reviewer: 평점/순위/출처/기준일
   └─ Image Reviewer: 저작권/중복/품질/alt·caption
6. 수정 반영
7. Editor Lead 최종 승인
8. 파일 반영
   └─ posts/{id}.js 생성
   └─ data/posts.js 메타데이터 추가
   └─ sitemap.xml 업데이트
9. 구문 검사
   └─ node --check posts/{id}.js
10. 커밋 & 배포
```

## 4. 포스팅 유형별 가이드

### 4-1. 추천형 포스트
- 검색의도 예: "넷플릭스에서 지금 뭐 볼까?"
- 필수: 취향/상황별 추천 분류 (데이트, 가족, 혼자, 밤, 가볍게)
- 필수: 각 작품에 대한 추천 이유 + 호불호 포인트
- 금지: 작품명만 바꾼 동일 구조 반복

### 4-2. 리뷰형 포스트
- 검색의도 예: "백 인 액션 볼만해?"
- 필수: 감상 포인트, 몰입도, 호불호, 추천 대상
- 필수: 로튼토마토/IMDB 등 외부 평점 + 출처
- 금지: 줄거리만 나열, 감상/판단 없음

### 4-3. 결말 해석형 포스트
- 검색의도 예: "브리저튼 시즌4 결말 무슨 뜻이야?"
- 필수: 스포일러 경고 명시
- 필수: 해석 근거 (장면, 대사, 원작, 감독 인터뷰 등)
- 필수: 다음 시즌/후속작 떡밥 정리

### 4-4. OTT 정리형 포스트
- 검색의도 예: "2026년 3월 넷플릭스 뭐 나와?"
- 필수: 월별/분기별 신작 목록 + 장르 분류
- 필수: 퇴장 예정작 정보
- 필수: 기준일 명시

### 4-5. 비교형 포스트
- 검색의도 예: "넷플릭스 vs 디즈니+ 뭐가 나아?"
- 필수: 비교표 (가격, 콘텐츠, 화질, 동접, 장단점)
- 필수: 상황별 추천 ("혼자 본다면 A, 가족이면 B")

### 4-6. 시리즈 가이드형 포스트
- 검색의도 예: "마블 뭐부터 봐야 해?"
- 필수: 시청 순서 (공개순/타임라인순)
- 필수: 건너뛰어도 되는 작품 / 필수 시청 작품 구분
- 필수: 소요 시간 안내

### 4-7. 트렌드형 포스트
- 검색의도 예: "지금 왜 오징어게임이 화제야?"
- 필수: 화제 이유 + 실제 추천 가치 판단
- 필수: 트렌드 데이터 (검색량, 넷플릭스 순위 등) + 출처
- 금지: 화제성만 좇고 내용 없는 빈 문서

### 4-8. 이관형 수정 (기존 티스토리 글)
- 기존 URL/핵심 키워드 기반 유지
- 내용 그대로 복사 불필요 — CLAUDE.md 품질 기준에 맞게 재구성
- AI형 도입부 제거 → 감정적 도입부로 교체
- 301 리다이렉트 매핑 필수

## 5. 이미지 가이드

### og:image 썸네일 (SNS/검색 공유용)
- 규격: 1200x630 SVG
- 내용: 카테고리별 배경색 + 큰 텍스트 (제목 + 부제 + 연도/출처)
- 파일명: `post{ID}_thumb.svg`
- 역할: og:image 전용, 본문에는 노출하지 않아도 됨

### 본문 이미지 (TMDB 크롤링)
- 소스: `scripts/crawl_movie_images.py --post-id {ID} --query "{영화이름}"`
- 파일명: `post{ID}_poster.jpg`, `post{ID}_still{N}.jpg`
- 모든 이미지에 `caption: '출처: TMDB'` 또는 `caption: 'ⓒ TMDB'` 필수

### 유형별 이미지 수량
| 글 유형 | 본문 이미지 |
|---|---|
| 단일 영화 리뷰 | 포스터 1 + 스틸컷 1~2 = 2~3장 |
| TOP N 추천 | 각 작품 포스터 1장씩 = N장 |
| OTT/비교형 | 주요 작품 포스터 2~3장 |
| OTT 비교 인포그래픽 | 가격/콘텐츠 비교표 SVG 추가 가능 |
| 시리즈 가이드 | 시청 순서 플로차트 SVG + 주요 작품 포스터 |
| 결말 해석 | 포스터 1 + 핵심 장면 스틸컷 1~2 |

## 6. 승인 실패 조건

아래 중 **하나라도** 해당하면 반려한다:

1. 영화/드라마/애니/OTT 외 주제
2. 날짜형/불릿형/이모지형 H2
3. 출처 없는 평점/순위 데이터
4. 광고 상한 초과 (RULE.md §4 참조)
5. 이미지 상한 초과(4장) 또는 저작권 위험 이미지
6. 내부링크 앵커가 날짜/카테고리 포함
7. body 5개 이하의 얇은 글
8. H2와 바로 다음 body 첫 문장 동일
9. intro에 감정적 도입 또는 핵심 판단 없음
10. 다른 글과 결론이 사실상 같은 복제 구조
11. AI형 도입 문구 사용
12. 줄거리만 나열, 감상/추천 이유 없음
13. 같은 문장 리듬이 다른 포스트와 반복

## 7. 신규 포스트 체크리스트

포스트 완성 후, 아래를 순서대로 확인한다:

```
□ node --check posts/{id}.js 통과
□ module.exports = post (CJS 방식)
□ data/posts.js에 메타데이터 추가됨
□ slug가 영문 키워드 기반 (핵심 키워드 2~4개 + 연도)
□ category가 5개 중 하나 (영화추천/드라마/해외반응후기/마블/애니메이션)
□ intro에 감정적 도입 + 핵심 판단 있음
□ "이번 포스팅에서는 ~알아보겠습니다" 없음
□ H2에 날짜/불릿/이모지 없음
□ 제목 앞쪽에 핵심 검색 키워드 배치됨
□ description이 핵심 키워드 전면 배치, 150자 이내
□ 광고 3개 이하 (장문 시 4개)
□ 이미지 수량이 글 유형 기준 이하 (단일 리뷰 4장, TOP N형 N장)
□ 본문 이미지는 TMDB 크롤링 포스터/스틸컷 사용
□ 출처 불명 이미지 없음
□ 이미지 alt/caption 구체적 + 출처: TMDB 표기
□ 평점/순위에 기준일/출처 있음
□ 내부링크 앵커가 설명형
□ body가 6개 이상
□ ending에 다음 작품/시리즈 안내 있음
□ 추천 대상과 비추천 대상이 구분되는가
□ 취향/상황/호불호가 같이 제시되는가
□ 줄거리 요약이 아니라 감상/선택 기준이 쓰여있는가
□ 같은 구조/문장 리듬이 다른 포스트와 반복되지 않는가
```

## 8. /posting_start 워크플로우

사용자가 `/posting_start` 또는 "포스팅 시작"이라고 하면 아래 절차를 실행한다.

### 절차

```
1. 주제/키워드 확인
2. 웹 크롤링으로 최신 정보 수집 (WebSearch, WebFetch)
3. SEO 키워드 전략에 따른 제목/slug/태그 설계
4. 포스트 파일 생성 (posts/{id}.js)
5. 이미지 크롤링 (scripts/crawl_movie_images.py --post-id {id} --query "{영화이름}")
6. 포스트에 크롤링된 이미지 삽입
7. data/posts.js 메타데이터 추가
8. node --check 구문 검사
9. 커밋
```

### 이미지 크롤링 규칙
- 단일 영화 리뷰: 포스터 1 + 스틸컷 1~2 = 2~3장
- TOP N 추천: 각 작품 포스터 1장씩 = N장
- OTT/비교형: 주요 작품 포스터 2~3장
- 모든 이미지에 alt + caption + 출처(TMDB) 표기

### SEO 최적화 규칙
- 제목: 핵심 검색 키워드를 앞쪽에 배치
- slug: 검색 키워드 2~4개 + 연도
- description: 핵심 키워드 전면 배치, 150자 이내
- tags: 핵심 + 롱테일 + 관련 키워드 6~8개
