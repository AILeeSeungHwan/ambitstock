## 3. 콘텐츠 규칙

### 구조
- 첫 2~3문단(intro) 안에 감정적 도입 + 핵심 판단 제시
- 각 글은 특정 검색의도 1개 또는 추천 상황 1개를 명확히 해결
- 취향·상황·호불호를 같이 제시

### SEO 구조 요소 (필수)
독자 체류 시간 및 검색 노출 품질을 높이기 위해 아래 요소를 **적극적으로** 활용한다.

| 요소 | 사용 상황 |
|---|---|
| `<ul><li>` | 특징 나열, 추천 대상, 체크리스트, 주요 포인트 |
| `<ol><li>` | 순위, 단계별 설명, 시청 순서 |
| `<table>` | 비교표, 요금제 표, 장단점, 스펙 비교 |
| `<blockquote>` | 실제 해외 리뷰 인용 (출처 명시 필수) |
| `<strong>/<b>` | 핵심 키워드, 작품명, 수치 강조 |

- 각 포스트에 ul/ol/table 중 **최소 2가지 이상** 사용
- body 블록 5개 이상의 긴 글은 **반드시 목록 또는 표를 1개 이상** 포함
- TOP N 추천글은 각 작품별 정보를 ul 또는 작품 카드 구조로 작성

### 톤앤매너 (일관성)
- **기준 톤**: 영화 좋아하는 친구가 솔직하게 알려주는 느낌, 직접적이고 구어체적
- **문체**: 현재형 위주. 과거 경험은 과거형 사용 가능. ~하였다는 ~했다로 줄여쓰기
- **금지 문체**: ~되었습니다 / ~하였다 / ~있었습니다 (과도하게 격식체인 경우)
- **권장**: ~입니다 / ~합니다 / ~있다 / ~이다 — 글 전체에서 체를 일관되게 유지
- **기준 포스트**: posts/480.js, posts/500.js, posts/509.js, posts/510.js (모범 예시)

### 사실 검증
- 평점(로튼토마토, IMDB), 시청 데이터, OTT 순위 → 기준일/출처 필수
- 추정치는 "추정치"로 명시

### 데이터 표현 규칙
아래 유형의 수치는 반드시 출처/기준일을 붙인다:
- 로튼토마토 평점, IMDB 점수, 관객수
- 넷플릭스 순위, OTT 시청 시간
- OTT 요금제 가격, 구독자 수

출처를 확인할 수 없으면:
- 수치를 삭제하거나
- "약 OO 수준으로 알려져 있습니다" 식으로 완화

### H2 규칙
- 사람이 읽는 소제목만 사용
- 금지: 날짜형, 불릿형(`-`로 시작), 이모지형(`📌`), 잘린 문장형
- H2와 바로 다음 body 첫 문장이 동일하면 안 됨

### 내부 링크
- 앵커에 날짜·카테고리·블로그 잔재 문구 금지
- 사람이 클릭하고 싶어지는 설명형 앵커 사용
- 같은 시리즈끼리, 같은 장르끼리 연결

## 4. 광고 규칙

| 조건 | 광고 상한 |
|---|---|
| 일반 포스트 (body ≤14) | 3개 |
| 장문 포스트 (body ≥15, h2 ≥6) | 4개 |
| 얇은 포스트 (body ≤5) | 2개 |
| 절대 상한 | 4개 |

### 배치 표준
```
intro → image → toc → [ad 상단] → h2/body... → [ad 중간] → h2/body... → ending → [ad 하단]
```

### 금지
- CTA 바로 앞뒤 광고 배치
- 이미지와 광고가 연속 (image → ad 또는 ad → image)
- body 없이 ad가 연속 (ad → ad)
- 총 5개 이상
- 첫 화면(스크롤 없이 보이는 영역)에 CTA/광고 우선 배치

## 5. 이미지 규칙

| 글 유형 | 이미지 기준 |
|---|---|
| 단일 리뷰/해석형 | 포스터 1 + 스틸컷 3~4 = **최소 4장** |
| 추천형 (TOP N) | 각 작품 포스터 1장씩 = N장 이상 (필수) |
| OTT/비교형 | 주요 작품 포스터 3~4 + 정보형 SVG 1 = **4~5장** |
| 트렌드형 | 대표 이미지 2~3 + 정보형 SVG 1~2 = **최소 4장** |
| 절대 하한 | **포스팅당 최소 4장** (이하 발행 금지) |

### 이미지 소스 규칙
- **본문 이미지**: 네이버에서 크롤링한 실제 포스터/스틸컷 사용 (`scripts/crawl_movie_images.py` 활용)
  - 명령어: `python3 scripts/crawl_movie_images.py --post-id {ID} --query "{작품명} 영화" --max-images 4`
- **og:image 썸네일**: 주제가 한눈에 보이는 큰 텍스트 + 클릭하고 싶은 색상 배합 (`post{ID}_thumb.svg`)
- 크롤링 이미지가 충분하지 않으면 다른 검색어로 재시도한다 (예: `"{배우명} 영화"`, `"{영문제목} movie"`)
- 모든 네이버 크롤링 이미지에 `caption: '출처: 네이버 영화'` 명시 필수
- **이미지 없는 포스트는 발행하지 않는다**
- **포스팅당 최소 4장 미만 발행 금지**

### 추천/비교 글 이미지 필수 규칙 (CRITICAL)
- **TOP N 추천 글**: 소개하는 작품마다 **반드시** 네이버 크롤링으로 포스터/스틸컷 이미지를 1장씩 삽입한다.
  - TOP 10이면 최소 10장, TOP 5이면 최소 5장
  - 각 작품 소개 h2/body 뒤에 해당 작품 이미지를 배치
  - `python3 scripts/crawl_movie_images.py --post-id {ID} --query "{작품명} 영화" --max-images 1` 반복 실행
- **비교 글 (A vs B)**: 비교 대상 작품/서비스마다 관련 이미지 1장씩 삽입
- 이 규칙은 예외 없이 적용한다. 이미지 크롤링에 실패한 작품이 있으면 다른 검색어로 재시도한다.

### 금지
- 네이버 외 출처 불명 포스터/스틸컷 사용
- 다른 글 메인 이미지 재활용
- 동일 이미지 반복
- 같은 배경 + 같은 폰트 + 제목만 교체하는 수준 (SVG 썸네일 한정)

### alt/caption
- alt: 구체적 설명 (나쁜: `영화 이미지` / 좋은: `2026년 3월 넷플릭스 신작 영화 라인업 총정리`)
- caption: 이미지가 왜 필요한지 + 출처 표기 (`출처: 네이버 영화`)

## 5-1. SEO 키워드 전략

### 제목 형식 (유형별)
독립 사이트는 블로그 플랫폼과 달리 도메인 자체에 검색 신뢰가 없으므로, 핵심 키워드를 제목 앞쪽에 배치해야 한다.

| 포스팅 유형 | 제목 패턴 |
|---|---|
| 리뷰형 | `[영화이름] 평점 출연진 후기 총정리 (연도)` |
| 추천형 | `[카테고리] 추천 [연도] 인기순 TOP N` |
| 해석형 | `[영화이름] 결말 해석 스포일러 떡밥 정리` |
| 비교형 | `[A] vs [B] 비교 [핵심 차이점] (연도)` |
| 가이드형 | `[시리즈명] 시청순서 입문 가이드 총정리` |

### slug 규칙
- 핵심 검색 키워드 2~4개 반영
- 연도 포함 권장 (시의성 있는 글의 경우)
- 예: `avengers-endgame-ending-explained-2026`, `netflix-top10-thriller-recommend-2026`

### description 규칙
- 핵심 키워드를 앞쪽에 배치
- 150자 이내
- 검색 스니펫 기준으로 작성 (독자가 클릭할 이유를 제시)

### tags 규칙
- 핵심 키워드 + 롱테일 키워드 포함
- 6~8개 권장
- 예: `['넷플릭스 추천', '2026 영화', '스릴러 추천', '넷플릭스 신작', '영화 추천', '공포 영화']`

## 5-2. 트렌드 포스팅 규칙

### tags 규칙
- 트렌드 포스트는 반드시 tags 배열에 `'트렌드'` 포함 (메인 배너 노출 조건)
- 예: `tags: ['트렌드', '넷플릭스 추천', '2026 영화', ...]`

### 메인 배너 노출 정책
- 트렌드 배너에는 `'트렌드'` 태그를 가진 최신 포스트 10개만 노출
- 오래된 트렌드 포스트는 날짜(date 필드) 기준으로 배너에서 자동으로 밀림
- 배너에서 밀린 포스트는 삭제되지 않으며 일반 포스트로 계속 유지됨

### 이미지 요건
- 트렌드 포스트는 이미지 없이 발행 불가
- 최소 구성: 썸네일 1 + 본문 이미지 2~3 + 정보형 SVG 1~2 = 최소 3장

## 7. 금지 문구

### H2에서 금지
- 날짜로 시작: `2025.09.19 - [영화] - ...`
- 불릿으로 시작: `- 넷플릭스 신작: ...`
- 이모지로 시작: `📌 스포일러 주의`
- 잘린 문장: `...결말이`

### 본문에서 금지 (맥락 없이 사용 시)
- `이번 포스팅에서는 ~에 대해 알아보겠습니다`
- `이번 글에서는 ~알아볼게요`
- `많은 분들이 궁금해하시는`
- `한눈에 정리해드리겠습니다`
- `지금 바로 확인하세요` (CTA 밖에서 남발)

### 과장·연출형 감정 표현 금지 (CRITICAL)
AI가 생성하는 감정 과장 표현을 일절 금지한다. 실제로 경험에서 나온 구체적 반응만 허용.

**절대 금지 표현:**
- `한동안 멍했다` / `한동안 말을 잃었다`
- `소름이 돋았다` / `온몸에 소름이` / `전율이 인다`
- `눈물이 났다` / `눈물을 참을 수 없었다`
- `가슴이 먹먹해졌다` / `숨이 막혔다`
- `심장이 쿵했다` / `심장이 내려앉았다`
- `충격적이었다` / `충격이었다` (근거 없이 사용 시)
- `말문이 막혔다` / `믿기지 않았다`
- `이렇게 좋은 영화가 있다니` 류의 막연한 찬사

**대신 이렇게:**
- 구체적으로 어느 장면/장치/연기 때문인지 서술
- "~라는 평가가 많다" / "로튼토마토 OO% / 관객 반응은 ~" (데이터 기반)
- "이 장면에서 연출의 의도가 느껴졌다" 등 구체적 서술

### 해외반응 포스팅 금지 사항
- **출처 없는 해외반응 작성 금지**: 인용할 반응은 실제 사이트(Reddit, IMDb, Rotten Tomatoes, Letterboxd 등)에서 확인된 것만 사용
- 실제 인용 시 출처를 괄호로 명시: `"(Reddit r/movies 반응)" / "(IMDb 사용자 리뷰 기준)"`
- 막연히 "해외에서 화제다" / "외국 반응이 엄청나다" 수준의 서술 금지
- 확인 불가 시 "OO기준 평점 OO점" 처럼 수치와 출처 기반으로만 서술

### 근거 없는 과장 표현 금지
- 무조건 추천 / 반드시 봐야 할 / 올해 최고 / 역대급
- 모든 사람이 좋아할 / 호불호 없는 명작
- (판단 근거가 있으면 사용 가능)

### 구조 금지
- 작품명만 바꾸고 결론이 같은 글
- 이미지 반복으로 문서 길이 늘리기
- 출처 없는 평점/순위 서술
- 광고 사이에 정보량이 거의 없는 얇은 문단
- 같은 문장 리듬이 3개 이상 포스트에서 반복

## 8. 파일 규칙

### 신규 포스트 파일 표준
```javascript
const post = {
  id: [번호],
  sections: [
    { type: 'intro', html: '...' },
    { type: 'image', src: '/images/post[ID]_thumb.svg', alt: '[구체적 설명]', caption: '[이유]' },
    { type: 'toc' },
    { type: 'ad', slot: '6297515693', format: 'auto' },
    { type: 'h2', id: 'section-id', text: '[사람이 읽는 소제목]', gradientStyle: 'linear-gradient(...)' },
    { type: 'body', html: '...' },
    // ... (body, h2, image 반복)
    { type: 'ad', slot: '6297515693', format: 'auto' },
    { type: 'ending', html: '...' },
    { type: 'ad', slot: '6297515693', format: 'auto' }
  ]
}
module.exports = post
```

### data/posts.js 메타데이터
```javascript
{
  id: [번호],
  slug: '[영문-키워드-slug]',
  title: '[한국어 제목]',

  category: '[영화추천|드라마|해외반응후기|마블|애니메이션]',
  date: 'YYYY-MM-DD',
  tags: ['태그1', '태그2', ...],
  thumbnail: '/images/post[ID]_thumb.svg',
  series: '[시리즈 식별자]',              // 선택
  relatedSlugs: ['slug1', 'slug2', ...]   // 선택
}
```

## 9. 커밋 규칙

```
[타입]: 설명
// 질문 의도: 한줄 요약
```

예시:
```
content: add bridgerton-season4 ending review post
// 질문 의도: 브리저튼 시즌4 결말 해석 포스팅 생성

migration: migrate 30 posts from tistory with 301 redirects
// 질문 의도: 기존 티스토리 포스팅 30개 이관 + 리다이렉트

fix: replace car insurance keywords in auto-post.sh
// 질문 의도: 자동 포스팅 키워드 그룹 영화/OTT용으로 교체
```
