## 3. 콘텐츠 규칙

### 구조
- 첫 2~3문단(intro) 안에 감정적 도입 + 핵심 판단 제시
- 각 글은 특정 검색의도 1개 또는 추천 상황 1개를 명확히 해결
- 취향·상황·호불호를 같이 제시

### 사실 검증
- 평점(로튼토마토, IMDB), 시청 데이터, OTT 순위 → 기준일/출처 필수
- 추정치는 "추정치"로 명시

### 데이터 표현 규칙
아래 유형의 수치는 반드시 출처/기준일을 붙인다:
- 로튼토마토 평점, IMDB 점수, 관객수
- 넷플릭스 순위, OTT 시청 시간
- OTT 요금제 가격, 구독자 수

출처를 확인할 수 없으면:
- 수치를 삭제하거나
- "약 OO 수준으로 알려져 있습니다" 식으로 완화

### H2 규칙
- 사람이 읽는 소제목만 사용
- 금지: 날짜형, 불릿형(`-`로 시작), 이모지형(`📌`), 잘린 문장형
- H2와 바로 다음 body 첫 문장이 동일하면 안 됨

### 내부 링크
- 앵커에 날짜·카테고리·블로그 잔재 문구 금지
- 사람이 클릭하고 싶어지는 설명형 앵커 사용
- 같은 시리즈끼리, 같은 장르끼리 연결

## 4. 광고 규칙

| 조건 | 광고 상한 |
|---|---|
| 일반 포스트 (body ≤14) | 3개 |
| 장문 포스트 (body ≥15, h2 ≥6) | 4개 |
| 얇은 포스트 (body ≤5) | 2개 |
| 절대 상한 | 4개 |

### 배치 표준
```
intro → image → toc → [ad 상단] → h2/body... → [ad 중간] → h2/body... → ending → [ad 하단]
```

### 금지
- CTA 바로 앞뒤 광고 배치
- 이미지와 광고가 연속 (image → ad 또는 ad → image)
- body 없이 ad가 연속 (ad → ad)
- 총 5개 이상
- 첫 화면(스크롤 없이 보이는 영역)에 CTA/광고 우선 배치

## 5. 이미지 규칙

| 글 유형 | 권장 이미지 수 |
|---|---|
| 리뷰/해석형 | 포스터 1 + 스틸컷 1~2 = 총 2~3장 |
| 추천형 (TOP N) | 각 작품 포스터 1장씩 = N장 (N이 10이면 10장+) |
| OTT/비교형 | 주요 작품 포스터 2~3 + 인포그래픽 1 = 3~4장 |
| 절대 상한 (단일 리뷰) | 4장 |
| TOP N형 상한 | N장 (초과 금지) |

### 이미지 소스 규칙
- **본문 이미지**: TMDB에서 크롤링한 포스터/스틸컷 사용 (`scripts/crawl_movie_images.py` 활용)
- **og:image 썸네일**: 기존 SVG 텍스트 이미지 유지 (`post{ID}_thumb.svg`)
- TOP N 형태의 포스트에서는 각 작품마다 포스터 이미지 1장씩 필수
- 리뷰/해석형 포스트에서는 포스터 1장 + 스틸컷 1~2장
- 모든 TMDB 이미지에 caption에 `ⓒ TMDB` 또는 `출처: TMDB` 명시 필수

### 금지
- TMDB 외 출처 불명 포스터/스틸컷 사용
- 다른 글 메인 이미지 재활용
- 동일 이미지 반복
- 같은 배경 + 같은 폰트 + 제목만 교체하는 수준 (SVG 썸네일 한정)

### alt/caption
- alt: 구체적 설명 (나쁜: `영화 이미지` / 좋은: `2026년 3월 넷플릭스 신작 영화 라인업 총정리`)
- caption: 이미지가 왜 필요한지 + 출처 표기 (`출처: TMDB`)

## 5-1. SEO 키워드 전략

### 제목 형식 (유형별)
독립 사이트는 블로그 플랫폼과 달리 도메인 자체에 검색 신뢰가 없으므로, 핵심 키워드를 제목 앞쪽에 배치해야 한다.

| 포스팅 유형 | 제목 패턴 |
|---|---|
| 리뷰형 | `[영화이름] 평점 출연진 후기 총정리 (연도)` |
| 추천형 | `[카테고리] 추천 [연도] 인기순 TOP N` |
| 해석형 | `[영화이름] 결말 해석 스포일러 떡밥 정리` |
| 비교형 | `[A] vs [B] 비교 [핵심 차이점] (연도)` |
| 가이드형 | `[시리즈명] 시청순서 입문 가이드 총정리` |

### slug 규칙
- 핵심 검색 키워드 2~4개 반영
- 연도 포함 권장 (시의성 있는 글의 경우)
- 예: `avengers-endgame-ending-explained-2026`, `netflix-top10-thriller-recommend-2026`

### description 규칙
- 핵심 키워드를 앞쪽에 배치
- 150자 이내
- 검색 스니펫 기준으로 작성 (독자가 클릭할 이유를 제시)

### tags 규칙
- 핵심 키워드 + 롱테일 키워드 포함
- 6~8개 권장
- 예: `['넷플릭스 추천', '2026 영화', '스릴러 추천', '넷플릭스 신작', '영화 추천', '공포 영화']`

| 역할 | 검수 항목 |
|---|---|
| SEO Reviewer | 검색의도 명확성, slug, 제목, 내부링크, 메타 구조 |

| Image Reviewer | 저작권 안전/중복/품질/alt·caption |
| Editor Lead | 최종 승인 — 아래 실패 조건 확인 |

### 승인 실패 조건 (하나라도 해당 시 반려)
1. 영화/드라마/애니/OTT 외 주제
2. 날짜형/불릿형/이모지형 H2
3. 출처 없는 평점/순위 데이터
4. 광고 상한 초과
5. 이미지 상한 초과 또는 저작권 위험 이미지 사용
6. 내부링크 앵커가 블로그 잔재 형태
7. body 5개 이하의 얇은 글
8. H2와 바로 다음 body 첫 문장 동일
9. intro에 감정적 도입 또는 핵심 판단 없음
10. 다른 글과 결론이 사실상 같은 복제 구조
11. "이번 포스팅에서는 ~알아보겠습니다" 등 AI형 도입 사용
12. 줄거리만 나열하고 감상/추천 이유가 없음

## 7. 금지 문구

### H2에서 금지
- 날짜로 시작: `2025.09.19 - [영화] - ...`
- 불릿으로 시작: `- 넷플릭스 신작: ...`
- 이모지로 시작: `📌 스포일러 주의`
- 잘린 문장: `...결말이`

### 본문에서 금지 (맥락 없이 사용 시)
- `이번 포스팅에서는 ~에 대해 알아보겠습니다`
- `이번 글에서는 ~알아볼게요`
- `많은 분들이 궁금해하시는`
- `한눈에 정리해드리겠습니다`
- `지금 바로 확인하세요` (CTA 밖에서 남발)

### 근거 없는 과장 표현 금지
- 무조건 추천 / 반드시 봐야 할 / 올해 최고 / 역대급
- 모든 사람이 좋아할 / 호불호 없는 명작
- (판단 근거가 있으면 사용 가능)

### 구조 금지
- 작품명만 바꾸고 결론이 같은 글
- 이미지 반복으로 문서 길이 늘리기
- 출처 없는 평점/순위 서술
- 광고 사이에 정보량이 거의 없는 얇은 문단
- 같은 문장 리듬이 3개 이상 포스트에서 반복

## 8. 파일 규칙

### 신규 포스트 파일 표준
```javascript
const post = {
  id: [번호],
  sections: [
    { type: 'intro', html: '...' },
    { type: 'image', src: '/images/post[ID]_thumb.svg', alt: '[구체적 설명]', caption: '[이유]' },
    { type: 'toc' },
    { type: 'ad', slot: '6297515693', format: 'auto' },
    { type: 'h2', id: 'section-id', text: '[사람이 읽는 소제목]', gradientStyle: 'linear-gradient(...)' },
    { type: 'body', html: '...' },
    // ... (body, h2, image 반복)
    { type: 'ad', slot: '6297515693', format: 'auto' },
    { type: 'ending', html: '...' },
    { type: 'ad', slot: '6297515693', format: 'auto' }
  ]
}
module.exports = post
```

### data/posts.js 메타데이터
```javascript
{
  id: [번호],
  slug: '[영문-키워드-slug]',
  title: '[한국어 제목]',
  description: '[검색 스니펫용 설명 150자 이내]',
  category: '[영화추천|드라마|해외반응후기|마블|애니메이션]',
  date: 'YYYY-MM-DD',
  tags: ['태그1', '태그2', ...],
  thumbnail: '/images/post[ID]_thumb.svg',
  series: '[시리즈 식별자]',              // 선택
  relatedSlugs: ['slug1', 'slug2', ...]   // 선택
}
```
