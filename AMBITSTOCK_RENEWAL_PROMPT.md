### 0-2. 핵심 문제점 (이번 리뉴얼에서 해결해야 할 것)

#### A. 홈 화면

| 문제 | 상세 | 영향 |
|------|------|------|
| Hero 영역 부재 | 사이트 정체성을 전달하는 Hero가 없음. 로고+메뉴 바로 아래 "지금 뜨는 작품" 시작 | 첫방문자가 "무슨 사이트인지" 파악에 3초 이상 소요 |
| 검색 기능 부재 | 589개 포스트에 검색 진입점 없음 | 원하는 작품/주제를 찾을 방법이 카테고리 스크롤뿐 |
| 카드 정보량 불균형 | "지금 뜨는 작품" 대형 카드에 설명문 2줄이 들어가나, "오늘의 트렌드"는 제목만 노출 | 같은 홈 안에서 카드 정보 밀도가 들쭉날쭉 |
| OTT 섹션이 링크 나열형 | 넷플릭스/디즈니+ 섹션이 텍스트 리스트로 되어 있어 시각적 탐색성 약함 | 유저가 OTT별 콘텐츠를 빠르게 훑기 어려움 |
| 카테고리 섹션 중복 노출 | 영화추천/해외반응후기/드라마/애니/마블이 각각 4개씩 카드로 나오는데 홈이 너무 길어짐 | 스크롤 피로, 하단 섹션 도달률 저하 |
| 콘텐츠 유형 배지 미분화 | "영화추천" 배지가 해석/추천/후기/정보 모두에 붙음 | 유저가 글 유형을 한눈에 구분 불가 |
| 하단 CTA 부재 | 홈 끝에 "전체 589개 포스팅 보기" 링크만 존재 | 재방문/구독/탐색 전환 유도 없음 |

#### B. 상세 페이지

| 문제 | 상세 | 영향 |
|------|------|------|
| 상단 정보 박스 없음 | 작품명/장르/플랫폼/핵심키워드 요약 없이 바로 본문 시작 | 유저가 "이 글이 어떤 작품에 대한 건지" 파악에 시간 소요 |
| 3줄 요약 없음 | 해석 글에서 "이 글에서 답하는 것"이 없음 | 글의 방향을 빠르게 파악 불가 |
| 스포일러 레벨 미표시 | 해석/결말 글에 스포일러 범위 표시 없음 | 유저가 스포일러 회피 판단 불가 |

| 중간 요약 블록 없음 | 긴 글에서 섹션 사이 핵심 포인트 정리 없음 | 긴 글 스캔성 부족 |
| CTA 부재 | 글 끝에 다음 행동 유도가 약함 | 이탈률 높음 |

#### C. 카드/썸네일 시스템

| 문제 | 상세 |
|------|------|

| 카드 변형 부족 | 대형/소형 2종류만 존재, 목록형/허브형/관련글형 변형 없음 |
| 요약문 길이 불통일 | 2줄~4줄까지 요약문 길이가 카드마다 다름 |

#### D. 내부링크 / 허브 구조

| 문제 | 상세 |
|------|------|
| 작품 허브 페이지 없음 | "프로젝트 헤일메리" 관련 글 4개가 있지만 묶는 허브 페이지 없음 |
| 주제 허브 없음 | "결말 해석 모음", "넷플릭스 추천 모음" 같은 주제 허브 없음 |
| 관련글 연결이 카테고리 기반 | 작품/주제 기반이 아닌 카테고리 기반 → 정밀도 낮음 |
| 상위 허브 링크 없음 | 상세 페이지에서 작품 허브/카테고리 허브로 올라가는 경로 없음 |

#### E. 카테고리 / 태그

| 문제 | 상세 |
|------|------|
| 카테고리 5개로 단순 | 영화추천/해외반응후기/드라마/애니메이션/마블 — "해석"이 독립 카테고리가 아님 |
| 태그 정규화 미완 | "넷플릭스 영화 추천", "넷플릭스 영화", "넷플릭스" 등 중복 태그 존재 |
| 태그 페이지 SEO 활용 없음 | 태그 페이지가 단순 글 목록 — 인트로/설명 없음 |
| 레거시 /entry/ URL 잔존 | 티스토리 시절 /entry/ 경로 글이 검색결과에 여전히 노출 |

#### F. SEO

| 문제 | 상세 |
|------|------|
| 사이트 주제 신호 약함 | 홈 메타 description이 "최신 영화, 드라마 추천!" 수준 |
| 허브 페이지 부족 | 작품/주제/OTT 허브가 없어 사이트 위계 신호 약함 |
| 목록 페이지 SEO 약함 | 카테고리 페이지에 인트로 텍스트/구조화 데이터 없음 |

| /entry/ 301 리다이렉트 미완 | 레거시 URL에서 새 URL로의 리다이렉트 누락 가능성 |

## PART 1. 리뉴얼 목표 (7개)

1. **블로그 → 영화 정보 사이트 전환**: 첫인상에서 "영화/드라마/애니 전문 사이트"로 인식
2. **탐색성 3배 향상**: 검색, 필터, 허브를 통해 원하는 콘텐츠를 3단계 이내에 도달
3. **콘텐츠 유형 가시성 확보**: 해석/추천/후기/해외반응/정보가 배지로 즉시 구분
4. **작품 허브 기반 구조 구축**: 개별 글 → 작품 단위 묶음 → 주제 허브 위계 확립
5. **카드 시스템 통일**: 썸네일 비율, 카드 변형, 정보 밀도를 규격화
6. **내부링크 구조화**: 글마다 같은 작품 글 / 같은 주제 글 / 상위 허브 연결 규칙 적용
7. **사이트 레벨 SEO 강화**: 허브 페이지, 구조화 데이터, 메타 통일, 태그 정규화

## PART 3. 홈 화면 리뉴얼 설계

### 섹션 순서 (위→아래)

```
1. GNB (로고 + 카테고리 메뉴 + 검색 아이콘 + 테마 토글)
2. Hero 영역
3. 지금 뜨는 작품 (대형 카드 1 + 소형 카드 5)
4. 무드별 추천 (가로 스크롤 칩 → 카드 전환)
5. OTT별 탐색 (넷플릭스/디즈니+/티빙/쿠팡 탭 → 가로 스크롤 카드)
6. 인기 콘텐츠 탭 (해석 | 추천 | 해외반응 | 드라마 — 탭 전환)
7. 작품 허브 하이라이트 (지금 반응 오는 작품 3~4개 허브 카드)
8. 최신 글 (3열 카드 그리드, 12개)
9. 하단 CTA (전체 보기 + 구독 유도)
10. 푸터
```

### 각 섹션 상세

#### 2. Hero 영역
- **배경**: 현재 화제작 스틸컷 블러 처리 또는 그라데이션
- **메인 카피**: "지금 볼 영화, 해석이 필요한 영화, 반응이 궁금한 영화까지"
- **서브 카피**: "영화·드라마·애니 추천부터 결말 해석, 해외반응까지 — R의 필름공장"
- **검색창**: 플레이스홀더 "작품명, 감독, 배우, 장르로 검색"
- **퀵 링크 3개**: 🔥 지금 인기 / 🧠 결말 해석 / 🌍 해외반응

#### 5. OTT별 탐색 (기존 개선)
- 현재: 텍스트 리스트 나열 → **변경**: 가로 스크롤 카드 (썸네일 + 제목 + 유형 배지)
- 탭: 넷플릭스 | 디즈니+ | 티빙 | 쿠팡플레이 | 극장
- 각 탭 3~6개 카드 + "전체 보기 →" 링크

#### 6. 인기 콘텐츠 탭 (기존 카테고리 섹션 대체)
- 현재: 영화추천/해외반응/드라마/애니/마블 각각 섹션으로 중복 노출 → **변경**: 탭 전환형 1개 섹션
- 탭: 해석 | 추천 | 해외반응 | 드라마 | 애니
- 각 탭 4개 카드
- **효과**: 홈 길이 60% 단축, 스크롤 피로 해소

## PART 4. 카드 시스템 설계

### 4-1. 콘텐츠 유형 배지 (기존 "영화추천" 1종 → 6종 분화)

| 배지 | 색상 | 적용 기준 |
|------|------|-----------|
| 해석 | #8B5CF6 (보라) | 결말 해석, Q&A, 복선 정리 |
| 추천 | #F59E0B (앰버) | 작품 추천, TOP N, 가이드 |
| 후기 | #10B981 (에메랄드) | 평점 정리, 관람평 |
| 해외반응 | #3B82F6 (블루) | 해외 평점, 해외 팬 반응 |
| 정보 | #6B7280 (그레이) | 출연진, 개봉일, 시청순서 |
| 분석 | #EF4444 (레드) | 흥행 분석, 시청률, 트렌드 |

### 4-2. 카드 변형 4종

| 카드 유형 | 용도 | 구성 |
|-----------|------|------|
| `CardLarge` | 홈 Hero 옆, 작품 허브 대표 | 16:9 썸네일 + 유형배지 + 제목(2줄) + 설명(2줄) + 날짜 |
| `CardMedium` | 홈 섹션, 카테고리 목록 | 16:9 썸네일 + 유형배지 + 제목(2줄) + 날짜 |
| `CardSmall` | 관련 글, 사이드바 | 4:3 소형 썸네일 + 제목(1줄) + 유형배지 |
| `CardList` | 검색 결과, 태그 페이지 | 좌측 소형 썸네일 + 우측 제목+설명+메타 |

### 4-3. 썸네일 규칙

- **비율 통일**: 모든 신규 썸네일 16:9 (1200×675px)
- **기존 poster.jpg (세로)**: `object-fit: cover`로 16:9 프레임에 맞춤
- **thumb.svg**: 16:9 기준으로 리사이즈
- **폴백**: 카테고리별 기본 썸네일 SVG 6종 준비 (영화/드라마/애니/마블/해외반응/분석)

### 4-4. 제목/설명 규칙

| 항목 | 규칙 |
|------|------|
| 제목 | 최대 2줄, `line-clamp-2`, 55자 내 |
| 설명 | 최대 2줄, `line-clamp-2`, 80자 내 |
| 날짜 | "3월 26일, 2026" 또는 "TODAY" 배지 |

## PART 5. 상세 페이지 리뉴얼

### 추가할 블록 (본문 위 → 아래 순서)

```
1. 브레드크럼 (홈 > 카테고리 > 작품허브 > 현재 글)
2. 유형 배지 + 날짜
3. H1 제목
4. 상단 정보 박스 (신규)
   - 작품명 | 장르 | 플랫폼 | 감독 | 출연
   - 스포일러 레벨: 🟢무스포 / 🟡약스포 / 🔴강스포
5. 3줄 요약 박스 (신규)
   - "이 글에서 다루는 것" 3줄
6. 목차 (기존 유지)
7. 본문 (기존 유지)
8. 중간 요약 블록 (긴 글에 삽입 — 신규)
   - 하이라이트 배경 + 핵심 포인트 2~3줄
9. 작품 허브 연결 (신규)
   - "[작품명] 더 보기" → 같은 작품 글 목록
   - 해석 | 후기 | 해외반응 | 정보 탭
10. 관련 글 (기존 개선)
    - 카테고리 기반 → 작품/주제 기반으로 변경
11. 다음에 읽을 글 CTA (신규)
12. 하단 태그
13. 광고 고지 + 푸터
```

## PART 7. 내부링크 구조 설계

### 글당 내부링크 규칙

```
각 포스트는 아래를 반드시 포함:
1. 같은 작품 글 → 최대 3개 (작품 허브 데이터 기반)
2. 같은 주제 글 → 최대 3개 (contentType 기반)
3. 상위 허브 → 1개 (작품 허브 또는 카테고리 허브)
4. OTT 허브 → 1개 (platform 기반)
5. 다음에 읽을 글 → 1개 (같은 작품 우선, 없으면 같은 주제)
```

### 자동화 방법

```typescript
// lib/internal-links.ts
export function getInternalLinks(currentPost: Post, allPosts: Post[]) {
  const sameWork = allPosts
    .filter(p => p.workSlug === currentPost.workSlug && p.slug !== currentPost.slug)
    .slice(0, 3);
  
  const sameTopic = allPosts
    .filter(p => p.contentType === currentPost.contentType && p.slug !== currentPost.slug)
    .sort((a, b) => b.date - a.date)
    .slice(0, 3);
  
  const parentHub = currentPost.workSlug 
    ? `/work/${currentPost.workSlug}` 
    : `/category/${currentPost.category}`;
  
  const ottHub = currentPost.platform 
    ? `/ott/${currentPost.platform}` 
    : null;
  
  return { sameWork, sameTopic, parentHub, ottHub };
}
```
