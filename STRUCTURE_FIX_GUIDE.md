## A. SVG 썸네일이 본문 sections에 포함 (544개)

### 현상
- `_thumb.svg` 파일이 sections 배열의 index 1 (intro 바로 뒤)에 image 섹션으로 포함됨
- 이 이미지는 og:image 전용으로, 본문에 렌더링되면 안 됨
- TOP N 포스트에서는 이 SVG가 1순위 이미지 자리를 차지하여 전체 이미지-순위 매핑이 1칸씩 밀림

### 해당 범위
- **1~544** (63 제외) + **559** = 총 544개
- 545~585 (559 제외)는 정상

### 수정 방법
각 포스트의 sections 배열에서 `_thumb.svg`를 src로 가진 image 섹션을 제거한다.

```javascript
// 제거 대상: 보통 sections[1]
{ type: 'image', src: '/images/post{ID}_thumb.svg', ... }
```

**주의**: 단순 제거만 하면 됨. og:image는 data/posts.js의 thumbnail 필드에서 관리됨.

### 자동 수정 스크립트 패턴
```javascript
post.sections = post.sections.filter(s => !(s.type === 'image' && s.src && s.src.includes('_thumb.svg')));
```
