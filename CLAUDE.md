## 5. 핵심 운영 원칙

1. **사용자가 "지금 뭘 볼지" 결정할 수 있게 써야 한다.** 글을 읽고 "이 작품이 나한테 맞는지 / 어디서 볼 수 있는지 / 비슷한 작품이 뭔지" 판단 가능해야 한다.
2. **줄거리 요약이 아니라 감상·추천·선택 기준을 써야 한다.**
3. **같은 템플릿을 복제하지 않는다.** 작품명만 바뀐 문단, 제목만 바뀐 구조, 같은 결론 반복 금지.
4. **취향·상황·호불호를 같이 제시한다.** "이런 사람에게 맞고, 이런 사람에겐 안 맞다"를 분명히.
5. **광고는 콘텐츠를 보조해야지 콘텐츠를 해치면 안 된다.**
6. **이미지는 네이버 크롤링 포스터/스틸컷을 본문에 사용한다.** SVG 텍스트 이미지는 og:image 썸네일 전용으로 유지한다. (`scripts/crawl_movie_images.py` 활용)
7. **홈은 단순 리스트가 아니라, 취향별 탐색이 가능한 추천 허브여야 한다.**
8. **트렌드에 반응하되, 빈 문서를 만들지 않는다.**

## 6. 포스트 구조 표준

모든 신규 포스트는 아래 구조를 기본으로 한다.

```
1. 한 줄 결론 — 이 작품을 볼지 말지 핵심 판단
2. 이런 사람에게 맞는 작품 — 취향/상황/분위기 기준
3. 스포일러 여부 / 감상 전 체크 포인트
4. 핵심 추천 이유 또는 해석 포인트
5. 분위기 / 장르 / 감정선 / 속도감 / 몰입도
6. 비슷한 작품과 비교
7. 이런 사람에게는 안 맞을 수 있음
8. 관련 작품/포스팅 내부 링크
```

### 섹션 작성 규칙
- `intro`: 2~3문단, 감정적 도입 + 핵심 정보. "이번 글에서는 ~알아보겠습니다" 금지.
- `toc`: 자연스러운 구조의 글에서만 사용
- `h2`: 사람이 읽는 소제목만 사용 (금지 패턴은 RULE.md 참조)
- `body`: 짧고 밀도 있게 작성. 한 body 블록 당 1~3 문단. 최소 200자.
- `image`: 본문에는 네이버 크롤링 포스터/스틸컷 사용. og:image 썸네일은 SVG 유지.
- `cta`: 본문 흐름을 해치지 않도록 후반부 1회 중심
- `ending`: 요약 + 다음 작품/시리즈 안내

## 8. 이미지 규칙

### 이미지 역할 분리
- **og:image 썸네일** (`post{ID}_thumb.svg`): SVG 텍스트 이미지 유지 — SNS/검색 공유용
- **본문 이미지**: 네이버에서 크롤링한 실제 포스터/스틸컷 사용 (`scripts/crawl_movie_images.py`)

### 썸네일/대표이미지 스타일 (og:image 전용)
**명료한 배경 + 큰 텍스트** 형식 (SVG 기반):

```
┌─────────────────────────────┐
│   [카테고리별 배경색/그라디언트]   │
│       큰 제목 텍스트           │
│       부제목/키워드            │
└─────────────────────────────┘
```

### 배경색 팔레트 (카테고리별)
| 카테고리 | 배경색 | 텍스트색 |
|---|---|---|
| 영화 추천 | #1a73e8 (파랑) | 흰색 + 노랑 강조 |
| 해외반응/후기 | #e53935 (빨강) | 흰색 |
| 마블 | #1a1a2e (다크네이비) | 흰색 + 골드 |
| 드라마 | #6a1b9a (보라) | 흰색 + 하늘색 |
| 애니메이션 | #00897b (틸) | 흰색 + 노랑 |
| 넷플릭스 | #e50914 (넷레드) | 흰색 |

### 이미지 규격
- og:image 썸네일: 1200x630 SVG, 파일명 `post{ID}_thumb.svg`
- 본문 실제 이미지: 네이버 크롤링 JPG/PNG, 파일명 `post{ID}_poster.jpg`, `post{ID}_still{N}.jpg`
- SVG 썸네일 텍스트: 최대 3줄 (제목 + 부제 + 출처/연도)

### 권장 수량
| 글 유형 | 본문 이미지 |
|---|---|
| 리뷰/해석형 | 포스터 1 + 스틸컷 1~2 = **2~3장** |
| TOP N 추천형 | 각 작품 포스터 1장씩 = **N장** |
| OTT/비교형 | 주요 작품 포스터 2~3장 = **2~3장** |
| 단일 리뷰 절대 상한 | **4장** |

### 금지
- 네이버 외 출처 불명 포스터/스틸컷 사용
- 다른 포스트의 메인 이미지 재사용
- 동일 이미지 반복
- 같은 배경 + 같은 폰트 + 제목만 교체하는 수준 (SVG 한정)

### alt/caption 규칙
- alt: 구체적 설명 (나쁜: `영화 이미지` / 좋은: `2026년 3월 넷플릭스 신작 영화 라인업`)
- caption: 이미지 설명 + 출처 표기 (`출처: 네이버 영화` 또는 `ⓒ 네이버 영화`)

## 9. H2 (소제목) 규칙

### 좋은 H2 예시
- `3월 넷플릭스 신작 중 가장 기대되는 작품 TOP 3`
- `혼자 밤에 몰입하고 싶다면 — 스릴러 추천`
- `시즌4 결말에서 놓치기 쉬운 복선 3가지`
- `로튼토마토 90% 넘긴 이유 — 해외 반응 정리`

### 금지 H2 패턴
| 패턴 | 예시 | 이유 |
|---|---|---|
| 날짜형 | `2025.09.19 - [영화] - 추천 목록` | TOC에 날짜 노출 |
| 불릿형 | `- 넷플릭스 신작: 레드 존` | 불릿이 소제목 역할 |
| 참고문구형 | `📌 스포일러 주의` | 이모지/운영메모가 소제목 |
| 잘린 문장형 | `...결말이` | 문장이 잘려서 끝남 |
| 본문 반복형 | h2 text와 바로 다음 body 첫 문장이 동일 | 중복 |

## 11. 콘텐츠 품질 기준

### 절대 원칙 (E-E-A-T)
1. **개인적 감상/의견 필수** — "이 장면에서 소름이 돋았다", "솔직히 기대 이하였다"
2. **경험 기반 서술** — "극장에서 직접 봤는데", "넷플릭스에서 3번 정주행한 입장에서"
3. **감정적 도입부** — "이 영화를 보고 나서 한동안 멍했다"
4. **구체적 데이터** — 로튼토마토 평점, IMDB 점수, 국내 관객수, 넷플릭스 순위
5. **독자 상황별 추천** — "데이트 영화로", "가족과 함께", "혼자 밤에 몰입하고 싶다면"

### 글의 톤
- 영화 좋아하는 친구가 카톡으로 추천해주는 느낌
- 스포일러 경고 명시 후 핵심 해석
- 반대 의견도 인정 ("호불호가 갈릴 수 있지만")
